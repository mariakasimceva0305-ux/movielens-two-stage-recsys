# MovieLens Two-Stage Recommender

Двухэтапная рекомендательная система на данных MovieLens. Решение разделяет быстрый отбор кандидатов и более точное переранжирование, что позволяет управлять качеством рекомендаций и задержкой ответа по отдельности.

## Назначение

По истории взаимодействий пользователя сначала отобрать небольшой набор кандидатов, а затем упорядочить его так, чтобы повысить качество верхних позиций и сохранить приемлемую задержку ответа.

## Архитектура

```text
История взаимодействий
        │
        ▼
Подготовка признаков
        │
        ▼
Отбор кандидатов
        │
        ▼
Модель переранжирования
        │
        ▼
Список рекомендаций top-K
        │
        ├─ расчёт метрик качества
        └─ API выдачи
```

## Что реализовано

- подготовка признаков по данным взаимодействий;
- отдельный этап отбора кандидатов;
- отдельный этап переранжирования;
- расчёт метрик качества рекомендаций;
- API для выдачи списка рекомендаций;
- сохранение артефактов обучения и оценки.

## Структура репозитория

- `src/train.py` — обучение компонентов системы;
- `src/candidate_generation.py` — логика отбора кандидатов;
- `src/features.py` — подготовка признаков;
- `src/ranking.py` — переранжирование;
- `src/evaluate.py` — расчёт метрик качества;
- `app/main.py` — API для выдачи рекомендаций;
- `artifacts/` — артефакты обучения и оценки;
- `notebooks/` — исследовательские проверки.

## Оценка качества

Основные метрики:

- `Precision@K`
- `Recall@K`
- `MAP@K`
- `NDCG@K`
- задержка ответа

Сравнение:

- отбор кандидатов без отдельного переранжирования;
- полный двухэтапный контур.

## Результаты

Численные метрики зависят от выбранного снимка MovieLens. После `python src/train.py --data-dir <каталог_с_ratings.csv_и_movies.csv>` смотрите:

- корневой `metrics.json` (сводка последнего обучения);
- `artifacts/runs/<run_id>/metrics.json` и `metrics.md` — копия отчёта с временной меткой;
- `artifacts/latest_run.txt` — идентификатор последнего сохранённого прогона.

Блоки `item_item`, `popularity` и `two_stage` в этом JSON относятся к одному прогону на выбранном снимке данных и позволяют сравнить контуры между собой.

## Запуск

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

python src/train.py --data-dir <путь_к_данным_MovieLens>
python src/evaluate.py --data-dir <путь_к_данным_MovieLens>

uvicorn app.main:app --host 0.0.0.0 --port 8000
```

## Ограничения

- метрики по историческим данным не заменяют проверку на реальном трафике;
- качество зависит от полноты истории взаимодействий и состава признаков;
- для устойчивого сравнения сохраняют результаты по нескольким вариантам моделей переранжирования.
