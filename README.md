# MovieLens Two-Stage Recommender

Двухэтапная рекомендательная система на данных MovieLens. Решение разделяет быстрый отбор кандидатов и более точное переранжирование, что позволяет управлять качеством рекомендаций и задержкой ответа независимо.

## Задача
Нужно по истории взаимодействий пользователя отобрать небольшой набор кандидатов, затем упорядочить его так, чтобы повысить релевантность верхних позиций и сохранить приемлемую задержку ответа.

## Архитектура решения
```text
История взаимодействий
        │
        ▼
Подготовка признаков
        │
        ▼
Отбор кандидатов
        │
        ▼
Модель переранжирования
        │
        ▼
Список рекомендаций top-K
        │
        ├──► Офлайн-оценка
        └──► API
```

## Структура репозитория
- `src/train.py` — обучение компонентов системы;
- `src/candidate_generation.py` — логика отбора кандидатов;
- `src/features.py` — подготовка признаков;
- `src/ranking.py` — переранжирование;
- `src/evaluate.py` — расчёт метрик качества;
- `app/main.py` — API для выдачи рекомендаций;
- `artifacts/` — сохранённые артефакты обучения и оценки;
- `notebooks/` — исследовательские проверки.

## Метрики
- `Precision@K`
- `Recall@K`
- `MAP@K`
- `NDCG@K`
- задержка ответа

## Результаты
В итоговую версию README нужно вставить фактические значения из артефактов оценки. Результаты должны включать:
- значения `Precision@K`, `Recall@K`, `MAP@K`, `NDCG@K`;
- сравнение двухэтапной схемы с упрощённым вариантом без отдельного переранжирования;
- задержку ответа для выдачи рекомендаций.

Формат записи:
- `Добавление этапа переранжирования повысило NDCG@10 с ... до ...`
- `Recall@10 увеличился с ... до ...`
- `Задержка ответа составила ... мс`

## Запуск
```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python src/train.py
python src/evaluate.py
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

## Ограничения и развитие
- Офлайн-метрики не заменяют проверку на пользовательском трафике.
- Качество зависит от полноты пользовательской истории и состава признаков.
- Следующий шаг — расширить набор признаков и добавить сравнение нескольких моделей переранжирования.
